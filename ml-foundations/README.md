# ml-foundations

Personal ML knowledge base organized by decision domain, not by algorithm. Written at practitioner depth: what each concept does, when it breaks, and how to fix it. Grounded in real project work (SickNote, DinoTracker, paleo-tech research). Not a textbook. Not a course. A reference I actually use.

## Folders

- **01-models-and-representations** — What different model types do and when to pick them
- **02-training-evaluation-debugging** — How to train, evaluate, read loss curves, and fix things when they break
- **03-data** — Splits, imbalance, leakage, augmentation, and distribution shift
- **04-paleo-tech** — Domain knowledge on ML applied to paleontology (footprints, CT scans, body mass)
